# Client site (Upriver scaffold)

Astro 5 hybrid static/SSR site generated by `upriver scaffold`. Public pages
are prerendered; `/admin/*` and `/api/*` run server-side on the Vercel
adapter.

## Requirements

- Node >= 22 (see `.nvmrc` / `package.json` `engines`)
- pnpm

```sh
pnpm install
pnpm dev        # local dev on http://localhost:4321
pnpm build      # production build (also generates dist/sitemap-index.xml)
pnpm typecheck  # astro check
```

## Environment

Copy `.env.example` to `.env` and fill in the values. Astro only exposes
`PUBLIC_`-prefixed vars to browser code; everything else stays server-only.

| Variable | Scope | Purpose |
| --- | --- | --- |
| `PUBLIC_SUPABASE_URL` | public | Client Supabase project URL |
| `PUBLIC_SUPABASE_ANON_KEY` | public | Anon key (RPC-only writes; RLS enforced) |
| `PUBLIC_SITE_URL` | public | Canonical origin — drives sitemap/canonical/OG URLs |
| `SUPABASE_SERVICE_ROLE_KEY` | server | Privileged reads/writes in `/admin` SSR routes |
| `GITHUB_TOKEN` / `GITHUB_OWNER` / `GITHUB_REPO` | server | Change-request form opens GitHub issues |

Apply `supabase/migrations/001_schema.sql` to the client's Supabase project
before first deploy.

## Admin access

`/admin/*` is gated by Supabase Auth (email + password) via `@supabase/ssr`.
There is **no public signup page** — create the first admin user out-of-band:

1. Supabase dashboard → Authentication → Users → **Add user** (set email +
   password, confirm the email), or
2. the operator's provisioning flow (service-role
   `auth.admin.createUser({ email, password, email_confirm: true })`).

Then sign in at `/admin/login`. Sessions are validated server-side with
`auth.getUser()` in `src/middleware.ts`.

## Forms

The scaffold provides three public form endpoints (POST, accepting
`application/x-www-form-urlencoded`, `multipart/form-data`, or JSON):

- `/api/inquiry` — contact / inquiry / RSVP forms → `inquiries` table via the
  `submit_inquiry` RPC; shows up in `/admin/inquiries`.
- `/api/newsletter` — signup forms → `newsletter_signups` table via the
  `subscribe_newsletter` RPC.
- `/api/donate` — records the submission as an inquiry (no payment
  processing).

Plain form POSTs are answered with a 303 redirect back to the referring page
with `?submitted=1`; JSON callers get JSON. A filled hidden `website` or
`_hp` honeypot field drops the submission silently. If Supabase env vars are
missing the endpoints return **503** — they never pretend success.

## Before launch

- Replace the operator-marked placeholder copy on `/privacy` and `/terms`.
- Point the `Sitemap:` line in `public/robots.txt` at the production domain.
- Set `PUBLIC_SITE_URL` to the production origin.
- Replace `public/favicon.svg` with the client's mark.
- Work through `launch-checklist.md` in the operator workspace
  (`upriver launch-checklist <slug>`).

## Monthly QA

QA does **not** run from this repo (`.github/workflows/monthly-qa.yml` is a
disabled stub). The operator runs it from the Upriver workspace, monthly:

```sh
upriver qa <slug> --preview-url <production-url>
```
